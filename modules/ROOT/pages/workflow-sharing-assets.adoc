:imagesdir: ../assets/images/

Anypoint Exchange provides the ability to publish assets in private Exchange instances and Exchange public portals. Publishing is the process of creating assets and making them available to consumers for discovery and reuse.

. xref:to-create-an-asset.adoc[Create an Asset] - Create new assets in Exchange.
. xref:to-describe-an-asset.adoc[Describe an Asset] - Add a description, additional pages, terms and conditions, and tags, and change versions.
. xref:lifecycle.adoc[Manage Asset Lifecycle States] - Manage the promotion path of an asset's lifecycle state.
. xref:manage-versions.adoc[Manage Versions] - View the version details of an asset, add a version for an asset, change the lifecycle state of a version, and delete a version.
. xref:to-manage-asset-portal-resources.adoc[Manage Asset Portal Resources] - List, retrieve, and delete resources from an asset portal with Exchange Experience API.
. xref:ex2-to-simulate-api-data.adoc[Simulate Data in REST APIs] - Use the mocking service to test OAS or RAML REST APIs with simulated data.
. xref:to-download-an-asset.adoc[Download an Asset] - Copy the asset to your computer.
. xref:to-manage-categories.adoc[Manage Categories] - Organize assets into categories.
. xref:to-manage-custom-fields.adoc[Manage Custom Fields] - Exchange administrators and organization owners can create custom fields to extend the Exchange asset metadata model and add custom data to any type of asset.
. xref:to-delete-asset.adoc[Delete an Asset] - Remove an asset.
. xref:to-change-raml-version.adoc[Change a RAML Version] - Update the version of a RAML asset.
. xref:publish-an-http-api.adoc[Publish an HTTP API Asset] - If you use Private Exchange, use the HTTP API to specify a placeholder for an endpoint.
. xref:add-policy-implementation-files-to-published-policy-definition.adoc[Add Policy Implementation Files to Published Policy Definition] - If you published a policy definition, add the implementation files to make policy available at runtime.
. xref:exchange2-faq.adoc[FAQ] - Frequently Asked Questions.
